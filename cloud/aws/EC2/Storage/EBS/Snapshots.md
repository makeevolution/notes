## Text Elements
EBS Snapshots ^2TcyNjB1

1. The first snapshot of a volume is always full, the next is incremental ^hk3LDfSG

2. You can use a snapshot to restore to a volume

But the loading is lazy (i.e. done as a background work)

To force the data to be directly loaded and immediately available in the volume, 
force read the block
using e.g. dd in Linux, or use FSR (Fast Snapshot Restore) feature
of AWS ^caOl9FMU

EBS snapshots are only done at used areas of a block
So your snapshots are charged purely based on used blocks ^BjzFSgVT
