## Text Elements
An EC2 instance is:
- A Virtual Machine (like VirtualBox)
- Thus, one host computer (called EC2 Host) can host multiple EC2 instances
- Attached to a subnet of a VPC ([[cloud/aws/VPC.md#^wR1jYYa6ABhWIb7UEyvhk|vpcsubnet]])
- If using Default VPC then we get public access configured for free ^dwsMSZDC

1. An instance lives in an availability zone
So if Availability zone fails, the instance fails.
An instance also has a temporary storage 
(instance store) ^GzAVsmjV

2. EC2 is provisioned to a subnet in a VPC ^HgLpYejo

3. This is a ENI (Elastic Network Interface).
An ENI in a subnet maps to a physical network interface in an EC2 instance ^MtQYu5zs

4. EBS is a service that provisions specific volume.
The volume also lives in a specific AZ! ^NheBQB1b

5. if an instance fails, it will reprovision
in the same AZ ^H18Az1zL

EC2 Instance Categories and Types
www.EC2Instances.info ^r8jthbGU

Picking an EC2 instance type must be depending on the use case you want ^JDdaseFa

How to connect to EC2 instances
- Usual SSH or Instance Connect
- Make sure your Security Group settings for these are configured safely
- For Instance Connect, it is actually an SSH connection from a EC2 instance service in AWS
- So, if we don't want all IP addresses in the internet to be able to SSH, need to set the IP of the service
- Same goes for if you only want your local laptop to be able to access through SSH ^S6g5jqQH

https://ip-ranges.amazonaws.com/ip-ranges.json

Use the IPs with service EC2_INSTANCE_SERVICE, and your Region ^SgYVenFr
