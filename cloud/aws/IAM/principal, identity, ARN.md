Identity:

- Anything that can be authenticated and then authorized

- In AWS, identities include:
1. IAM Users (human accounts with long-term credentials).

2. IAM Roles (assumed by AWS services, apps, or users with temporary credentials).

3. Federated Identities (external users via SSO, Cognito, or external IdPs like Google, Okta, AD).

4. IAM Groups (collection of users, though groups can’t be assumed directly). ^m2b539Qz

ARN (Amazon Resource Name):
- Format is arn:partition:service:region:account-id:resource

arn        Literal prefix to indicate it’s an ARN
partition        AWS partition: usually aws (other options: aws-us-gov, aws-cn)
service        AWS service: s3, lambda, ec2, etc.
region        Region of the resource: us-east-1, eu-west-1. Some global services like IAM or s3 don’t have a region (empty).
account-id        AWS account number (12 digits)
resource        Resource identifier (can include type and name/ID)

- Examples
arn:aws:s3:::my-bucket
arn:aws:lambda:us-east-1:123456789012:function:MyLambdaFunction
arn:aws:iam::123456789012:role/MyLambdaExecutionRole
arn:aws:ec2:us-east-1:123456789012:instance/i-0123456789abcdef0
