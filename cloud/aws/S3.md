Unique ID stored in a Bucket ^IOrTRxMx

Bucket
- Container that stores objects
- Configured in a region [[cloud/aws/VPC.md#^wR1jYYa6ABhWIb7UEyvhk|vpc]]
- Bucket names are globally unique (no one else in the world can use your name)
- Flat structure (no file structure) ^6QClatB2

More details:
- S3 is an object store - not file or block
- Thus you cannot mount an S3 bucket as C:\ for example
- When you make a folder in the UI and upload a file into it, it doesn't make a folder
  but makes an object with name: folder/filename







- A bucket is a resource; it has a unique Amazon Resource Name (ARN)

Versioning:
- If a bucket is not configured with versioning, file uploads with the same name
will overwrite the existing one ^odFzLS8W

this is fake; S3 actually creates objects with name 
test/filenameInTest1, test/filenameInTest2, etc. ^A3yaRAGh
