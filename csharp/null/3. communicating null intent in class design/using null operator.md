Dependency (without nullable context set) ^sYVg2wFw

Client ^YBcEolNu

1. Notice that the compiler of the client, even with its nullable enabled, still doesn't help.
Address still is said as not null, although p.Address is indeed null!
We will get NullPointerException! ^KHrWq3aO

2. This is because the nullable context is false in the dependency. The compiler then in this case assumes that the intent of the Property is that the Address is not null (so this is like the null forgiving operator) ^1DFp256m

3. With the context enabled, now, in the dependency we are forced in to fix this issue.

But notice that if we don't fix it, the client is still in danger (still no warning here)! So don't ignore this! ^5L06gvUn

4. With nullable context, and clear intention declaration, the compiler and the client is informed correctly ^RC7V6BdA

Let's say we are designing a Person class, where the intention is that a person can have no address i.e. homeless.
If we blindly just implement the POCO without setting nullable context, and declaring intention, then the client code may broke! ^nJV8egoW
