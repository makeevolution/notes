When assigning a variable to result of another operation that returns reference type, 
also specify your null intent; i.e. don't use var ^YnePXOBX

Be careful, the null conditional operator (i.e. ?) does not guard you against empty values!
So you will still get error here: ^GHZIvCHD

Solution: To safeguard against empty arrays, access the element using LINQ
It will return the default of the type the array contains (e.g. for int it will be 0)

Notice that the intent is clear already through the assignment

Don't use null forgiving, throw exception/do null checks there instead ^ea10humg

Null forgiving is dangerous since it is false confidence; it turns off all compiler warnings.
If someone else were to change a line of code before the line where the null forgiving is used,
this change may cause the null forgiving to let a bug through!
 ^4K4viPc1

If, during a refactor session let's say, someone were to comment this, 
app will be null, although we are confident
that it will not be ^VmFsjkAV

A better approach is, to throw an exception there instead of being confident
Throw exception instead of null check, since it being not null is an exception i.e. bug, not a expected event that we know how to handle e.g. invalid end-user input
